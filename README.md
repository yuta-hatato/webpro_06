# Webプログラミング演習 レポート要約

## 1．概要
本プロジェクトは，Express（Node.js）を使用して構築されたAPIサーバと，fetch APIを用いた静的Webページを統合したアプリケーションである．千葉県市町村，千葉工業大学施設，NPB球団という3つの異なるテーマを扱いながら，共通の設計思想に基づき「一覧から詳細へ」という統一的なユーザー体験を提供している．

## 2．各アプリケーションの構成と工夫点
### 2.1 千葉県市町村一覧
* **多角的な視点**: 利用者・管理者・開発者の各ニーズに応える設計となっている．
* **情報の重み付け**: 山武市の情報を他自治体より充実させることで，特定のデータに注目させるUI/UXを実現している．
* **保守性**: データは`items.json`で一元管理されており，ソースコードを書き換えることなく情報の更新が可能である．

### 2.2 千葉工業大学施設一覧
* **利便性**: 新入生や来訪者をターゲットとし，キャンパス内の主要施設情報を網羅している．公式サイトへのリンクを含むなど，実用的な構造となっている．

### 2.3 NPB球団一覧
* **機能性**: リーグ別フィルタリングや検索機能を実装し，目的の情報へ素早くアクセスできる．
* **独自データ**: 「最後に日本一になった年」などの付加情報をバッジ表示し，データの視認性を高めている．



## 3．共通設計・技術スタック
* **バックエンド**: ExpressによるREST風APIの構築（エンドポイントの共通化）．
* **フロントエンド**: HTML/JavaScript（fetch API）による動的なDOM操作．
* **データ管理**: データベースの代わりにJSONファイルを採用した軽量な構成．
